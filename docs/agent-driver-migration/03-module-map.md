# Module Map

| Migration Unit | Module | Role | Responsibility | Inputs | Outputs | Portability | Notes |
| --- | --- | --- | --- | --- | --- | --- | --- |
| `core-contracts` | `agent-driver-v2/src/core/interfaces.ts`, `agent-driver-v2/src/index.ts` | `contract` | Define public protocol for sources, config, tool results, and package exports | AI SDK message/tool types, host callback contracts | TypeScript interfaces and public entry points | `portable` | Foundation for every other bundle |
| `run-loop-orchestrator` | `agent-driver-v2/src/core/agent-driver-v2.ts`, `agent-driver-v2/src/utils/debounce.ts`, `agent-driver-v2/src/utils/logger.ts` | `orchestrator` | Coalesce updates, collect messages/tools, normalize protocol, route tools, manage state machine | `IDrivenSource[]`, `LLMClient`, callbacks | assistant/tool events, streaming deltas, state transitions | `portable` | Highest-value transplant unit |
| `llm-provider-bridge` | `agent-driver-v2/src/core/llm-client.ts`, `agent-driver-v2/src/core/provider-factory.ts` | `adapter` | Normalize model IDs, build provider registry, guard unsupported attachments, call `streamText`, fallback for OpenRouter tool issues | `LLMConfig`, messages, tools, env vars | `LLMResponse`, usage, tool calls | `adapt` | Portable if target also uses AI SDK v6; otherwise rewrite |
| `package-tests` | `agent-driver-v2/tests/core/*.test.ts`, `agent-driver-v2/vitest.config.ts`, `agent-driver-v2/tsconfig.json` | `test-support` | Preserve invariants around ordering, tool routing, protocol-safe history, and state semantics | mock sources, driver internals | executable regression tests | `portable` | Current suite exposes one important risk: some tests accidentally require live provider config |
| `system-prompt-source` | `host/src/adapters/system-prompt-source.ts` | `adapter` | Inject timestamp-0 system prompt as a regular source | system prompt string | one system message + update notifications | `portable` | Simplest source pattern and easiest first adapter |
| `runtime-aotui-source` | `runtime/src/adapters/aotui-driven-source.ts`, `runtime/src/adapters/system-instruction.ts`, `runtime/src/adapters/index.ts`, `runtime/src/adapters/aotui-driven-source.test.ts` | `adapter` | Convert AOTUI desktop snapshots and operations into driver messages/tools | desktop snapshots, kernel operations, instruction text | messages, tools, tool execution, update events | `adapt` | Strong reference bundle if target has desktop/view/tool state |
| `host-history-source` | `host/src/adapters/host-driven-source.ts`, `host/src/core/message-service-v2.ts` | `adapter` | Surface persisted host history and optional context compaction as a source | message storage, compaction policy | history messages, `context_compact` tool, update events | `adapt` | Good pattern for any host-owned message backlog |
| `im-history-source` | `host/src/im/im-driven-source.ts` | `adapter` | IM-specific history source with compaction anchors and synthetic messages | IM session history + compaction summary | compacted history, compaction tool messages | `adapt` | Optional but valuable if target supports long-lived chat threads |
| `host-session-composition` | `host/src/core/session-manager-v3.ts`, `host/src/core/host-manager-v2.ts` | `orchestrator` | Compose sources, resolve model config, connect callbacks to persistence and GUI, manage session lifecycle | desktop manager, kernel, config, message service, sources | live session with driver + UI events | `rewrite` | Use as reference architecture, not drop-in code |

## Deliberate Exclusions

- `SkillDrivenSource` and `McpDrivenSource` are adjacent extensions, not part of the minimum `agent-driver-v2` migration set.
- DesktopManager, DB services, and topic/project stores are host infrastructure, not core feature code.
